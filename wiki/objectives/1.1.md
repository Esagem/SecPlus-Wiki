# 1.1 — Security Controls

CompTIA description: *"Compare and contrast various types of security controls."*

The whole objective is about classifying controls along two axes — the **category** (how it's implemented) and the **type** (what it does in the event lifecycle). Master the matrix in [[concepts/security-controls|Security Controls overview]] and you've got the objective.

## Concepts

### The framework

- [[concepts/security-controls|Security Controls]] — overview + the 4×6 matrix

### Control types (the 6 things a control can *do*)

- [[concepts/preventive-controls|Preventive]] — blocks
- [[concepts/deterrent-controls|Deterrent]] — discourages
- [[concepts/detective-controls|Detective]] — identifies
- [[concepts/corrective-controls|Corrective]] — undoes after the fact
- [[concepts/compensating-controls|Compensating]] — substitutes for an unavailable control
- [[concepts/directive-controls|Directive]] — instructs (weakest)

### Control categories (the 4 ways a control can be *implemented*)

Covered inside the overview page. The four are:

- **Technical** — done by systems / software
- **Managerial** — done by policy
- **Operational** — done by people
- **Physical** — done by tangible barriers
