# 2.2 — Threat Vectors and Attack Surfaces

CompTIA description: *"Explain common threat vectors and attack surfaces."*

A **threat vector** (or attack vector) is the *path* an attacker takes to get in. Objective 2.1 was about *who*; this objective is about *how*. The exam tests two clusters: the technical/structural vectors (messages, files, USB drops, default creds, supply chain) and the social-engineering vectors that target the user rather than the system.

---

## The two halves of the objective

| Cluster | What it targets | Examples |
|---|---|---|
| **Technical / structural vectors** | The system, the network, the supply chain | Open ports, default credentials, unsupported systems, wireless, USB, vulnerable software, MSP compromise |
| **Human / social-engineering vectors** | The user's trust, attention, or hurry | Phishing, vishing, smishing, impersonation, pretexting, watering holes, misinformation |

Most real-world attacks chain them — a phishing email (human vector) delivers a malicious macro (file vector) that pivots through an MSP (supply-chain vector). Scenario questions almost always describe the *entry point*; identify that, and the answer follows.

---

## Message-based vectors

The single most common entry point because everyone uses messaging.

| Channel | Common attack | Notes |
|---|---|---|
| **Email** | Phishing, malicious attachments, malicious links, invoice scams, crypto scams | The default vector for almost every campaign |
| **SMS** | Smishing — "your package is delayed, click here" | Trusted on mobile devices, often less filtering than email |
| **IM / DM** | Direct social engineering, link delivery | Bypasses email gateways entirely |

**Exam trap:** the message itself is the *vector*; what arrives — phishing, malware, fraud — is the *attack*. A question describing "USPS package suspended, click to verify address" is asking about the SMS vector used to deliver smishing.

### Image-based vectors
An image file can be a vector when the format permits embedded code. The classic exam example is **SVG** (Scalable Vector Graphics) — an XML-described image that can include JavaScript or HTML. When the browser renders the SVG, it can execute embedded script. Most modern browsers harden against this, but unvalidated SVG input is still a recognized vector.

### File-based vectors
Any file format that can carry code or be executed is a vector:

- **Executables** (.exe, .msi, .app)
- **PDF** — embedded scripts and objects
- **Office documents** — macros (Word, Excel, PowerPoint)
- **Compressed archives** (.zip, .rar) — obfuscates the malicious file inside
- **Browser extensions / add-ins** — full access to browser session

**Exam trap:** "the executable is the vector" is wrong if the question highlights how it *arrived*. The vector is whatever path delivered it (email attachment, USB drop, drive-by download).

## Voice and call vectors

| Term | What it is |
|---|---|
| **Vishing** | Voice phishing — calling a target to extract credentials, financial info, or get them to take an action |
| **SPIT** | Spam over IP — automated robocall spam delivered via VoIP |
| **War dialing** | Sweeping ranges of phone numbers looking for modems, fax-backs, or other reachable systems |

---

## Removable-device vectors

The classic $10 USB drop attack bypasses millions of dollars of network security. Three flavors matter:

- **USB drop** — leave the drive in a parking lot, hope someone plugs it in. Especially effective for bridging air-gapped networks.
- **HID injection (BadUSB)** — a specially modified USB device that registers as a keyboard and types attacker commands automatically when plugged in.
- **Data exfiltration** — small device, large capacity, walks out the door in a pocket.

**Exam trap:** any answer mentioning "air-gapped" environments usually points to USB or other removable-device vectors. No network connection → physical media is the path in.

## Software and service vectors

### Vulnerable software
- **Client-based** — software installed locally; you have to keep every endpoint patched.
- **Agentless / web-based** — application lives on a server, users connect via browser. If you compromise the server, every client running the app today is exposed.

### Unsupported systems
End-of-life operating systems and applications receive no further security patches. Each one is a permanent open door until removed or compensating controls are deployed. Discoverable via asset inventory and periodic scans.

### Open service ports
Every running service on a host opens a port. More services = larger attack surface. Mitigation: port-based firewalls, application-aware firewalls, disable unused services.

### Default credentials
Routers, IoT, IP cameras, admin consoles — many ship with documented usernames and passwords (`admin`/`admin`). Sites like routerpasswords.com catalog them publicly. The Mirai botnet exists almost entirely because of unchanged default credentials on IoT devices.

---

## Network vectors

- **Wireless** — outdated protocols (WEP, WPA, WPA2 in some configurations) and rogue access points provide easy entry. Move to **WPA3**.
- **802.1X** — port-based network access control; requires authentication before allowing any network access, wired or wireless.
- **Bluetooth** — used for reconnaissance and exploitation when implementations lack proper security.

---

## Supply-chain vectors

Equipment and services you trust become vectors when they themselves are compromised.

- **Hardware supply chain** — counterfeit equipment with backdoors or malware (e.g., counterfeit Cisco switches uncovered in 2022; counterfeit Catalyst switches in 2020).
- **Managed Service Providers (MSPs)** — third-party with privileged access to your systems. Compromise the MSP, compromise everyone they manage.
- **Vendors with network access** — the **Target breach (2013)** used an HVAC contractor's network access as the pivot to 40M+ credit-card records.
- **Software supply chain** — covered in depth in [[objectives/2.3|2.3]] (SolarWinds Orion).

**Exam trap:** "supply chain" on the exam covers both physical hardware AND services/vendors with system access. If a question describes how a third party with legitimate access was used to reach the target, that's a supply-chain vector regardless of whether any "supply" was physically delivered.

## Social engineering — the human vectors

### Phishing (email)
Email-based social engineering. Tries to get the victim to click a malicious link, open an attachment, or surrender credentials on a fake login page.

Common tells:
- Mismatched sender domain (`professormesser.com` vs. `professormessor.com`)
- Hovering a link reveals an unexpected destination
- Subtle font/spacing problems on cloned login pages
- Urgency or deadline pressure ("act within 24 hours")

### Vishing (voice phishing)
Same goal, voice channel. Often spoofs caller ID to appear to come from a bank, IRS, Microsoft support, etc.

### Smishing (SMS phishing)
Same goal, text channel. The classic example: "USPS: your package can't be delivered, click here to update your address."

### Pretexting
The *story* the attacker tells to make the social engineering work. "Hi, this is John from IT — there's a problem with your account, I need your password to fix it." Pretexting is the wrapper around all the channel-specific attacks above.

### Impersonation
Pretending to be someone the victim trusts or fears — IT help desk, the CFO, a vendor, law enforcement. Two flavors:

- **Authority impersonation** — pretending to be senior or institutional ("VP of Finance," "IRS agent") to short-circuit normal scrutiny.
- **Peer impersonation** — pretending to be someone in the same org ("I work in the same department") to gain trust.

Closely related concepts the exam treats under impersonation:
- **Eliciting information** — extracting useful details through casual-seeming questions, without ever asking for the sensitive item directly.
- **Identity fraud** — opening accounts, filing taxes, or taking loans in the victim's name. The crime that often follows successful impersonation.

### Typo squatting / URL hijacking / brand jacking
Registering domains that look almost like the legitimate one — `professormessor.com`, `professormeser.com`, `professormesser.org`. Catches users who mistype, misread, or click without verifying.

### Business Email Compromise (BEC) — the spoofed sender
Email that appears to come from an executive, vendor, or trusted internal address, often requesting a wire transfer or sensitive document. Sometimes uses a lookalike domain; sometimes uses a compromised real mailbox.

### Watering hole attacks
The attacker doesn't attack the target directly — they attack a third-party site the target *visits regularly* (the "watering hole"), and wait. Real example: in 2017, the Polish Financial Supervision Authority's website was infected with JavaScript that only activated against specific bank IP ranges. Defense is depth: AV + firewall + IPS so one layer catches what another misses.

### Misinformation, disinformation, influence campaigns
Factually incorrect content disseminated to divide, confuse, or persuade — often via social media, often by foreign actors. The mechanism:

1. Attacker creates fake user accounts.
2. Posts content from those accounts.
3. Other fake accounts amplify (like, share, follow).
4. Algorithm picks up the artificial engagement and shows real users.
5. Real users share it. Eventually mass media notices the "trend" and covers it.

### Brand impersonation in search/ads
Attackers register thousands of sites containing recognizable brand names, then rely on search ranking or paid ads to lure clicks. Lookalike landing pages trick the user into downloading malware or entering credentials.

## How to read scenario questions

1. **Identify the channel** — email, SMS, voice, USB, network, supply chain, image/file. The channel is usually the vector the question is testing.
2. **Identify the lever** — is the attack technical (an unpatched port, default credential) or social (trust, urgency, authority)? Social-engineering questions name a feeling; technical-vector questions name a configuration.
3. **For social-engineering questions:** match the medium to the term — email → phishing, voice → vishing, SMS → smishing, in-person/by-pretext → impersonation.
4. **For supply-chain questions:** look for any third party — vendor, MSP, contractor, manufacturer. If the third party was used as the pivot, the vector is supply chain even if the third party's role seems unrelated (HVAC contractor → cash registers).
