# Sessions

Dated study session notes. Filename: `YYYY-MM-DD-<short-slug>.md`. One file per session.

See the template in [[STUDY|STUDY §4]].

## What a session note captures

- **Covered** — which objectives and concepts were worked.
- **Quiz result** — score, if a quiz was run.
- **Missed** — what specifically you got wrong, in your own words.
- **Updated** — which concept pages had their mastery field flipped, which other pages were touched.
- **Next** — the concrete next step. "Re-quiz 2.4 in 3 days." "Read X concept before next session."

Sessions are the durable trace of how your understanding evolves. They're the source data for `synthesis/weak-areas.md` and for spotting which concepts have been re-missed too many times.
