# Concepts

Atomic concept pages. One concept per page. Filename: `kebab-case.md`.

Each page must include:
- `objective:` front-matter field mapping to one or more exam objectives (e.g. `"2.3"`).
- A one-line definition near the top.
- A "Related" section linking other concept pages.
- A "Source" section pointing back to the transcript section(s) it was distilled from.

See the template in [[STUDY|STUDY §4]].

## Convention notes

- **Atomicity over completeness.** It's better to have 200 short concept pages than 30 long ones — easier to cross-link, easier to quiz against.
- **Self-mastery in the YAML.** Use `mastery: weak|building|strong` so study sessions can target weak ones.
- **No transcript copy-paste.** The point of a concept page is the distilled understanding — facts, mechanism, defenses, traps. Quoting more than a sentence from the transcript means the page isn't doing its job.
